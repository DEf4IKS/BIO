Правила использования MinamotoSoft

MinamotoSoft — мощный инструмент для управления множеством Telegram-аккаунтов. Простой интерфейс, продвинутые возможности — идеальное решение как для новичков, так и для опытных пользователей.

1.2. Любое копирование, распространение или модификация программы без письменного согласия разработчика запрещены.
1.3. Пользователь обязан соблюдать все действующие законы и правила Telegram и своей страны при использовании программы.
1.4. Все действия, связанные с нарушением правил и законов, выполняются пользователем на свой страх и риск; разработчик не несёт ответственности за последствия противоправных действий.

2. Ответственность пользователя

2.1. Пользователь несёт полную ответственность за любые операции, совершаемые через программу, включая массовые подписки, рассылки, участие в конкурсах и другие автоматизированные действия.
2.2. Пользователь обязуется использовать программу в правовых и этичных целях.

3. Управление аккаунтами

3.1. Пользователь может изменять имя, настраивать приватность и управлять подписками своих аккаунтов.
3.2. Добавление аккаунтов возможно через QR‑код, код подтверждения или импорт существующих сессий (при наличии прав на эти сессии).
3.3. Программа позволяет централизованно просматривать входящие сообщения и управлять чатами без переключения между аккаунтами.

4. Использование модулей

4.1. Пользователь может подключать модули для автоматизации задач, в том числе:
- Subscribe / Unsubscribe
- Referrals (автоматическое участие в конкурсах и решение капч)
- Run button (автоматическое нажатие кнопок)
- Giveshare, Random1ze, Telecombine
- Cryptobot checks, Ping, Stars
- e (выполнение Python-кода внутри программы)
4.2. Автоматизация должна применяться только для легальных и добросовестных действий. Запрещается использование модулей для спама, мошенничества, распространения вредоносного ПО или других противоправных действий.

5. Финансовые операции

5.1. Программа позволяет отслеживать баланс аккаунтов (например, через @CryptoBot).
5.2. Пользователь несёт ответственность за финансовые операции и любые последствия, связанные с использованием средств.

6. Системные требования и логирование

6.1. Все действия программы логируются для отладки и безопасности.
6.2. Логи используются для выявления ошибок и контроля работы аккаунтов.
6.3. Разработчик не несёт ответственности за блокировки или ограничения аккаунтов, если эти действия вызваны нарушением правил Telegram или злоупотреблениями со стороны пользователя.
